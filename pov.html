After some Googling, I learned that ViewState is another ASP.NET feature for preserving page state between postbacks. They are serialized into base64-encoded strings and stored as hidden form values, which will be included in each postback request. You can read more about this in Microsoftâ€™s documentation.

I also came across this blog post, where it details how ViewState can be vulnerable to deserialization attacks. Under certain circumstances, it may be possible to forge a malicious ViewState value and gain RCE on the server.
