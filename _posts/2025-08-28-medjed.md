excerpt: Recently I’ve been doing more Proving Grounds in preparation for my OSCP exam later this year. These boxes certainly have a different “flavour” than HTB ones, often containing many different applications and services running simultaneously. OffSec also loves chaining multiple applications together in an exploit chain. Finding a vulnerability in an app often does not lead to a foothold directly, but instead is used to attack or enumerate other services. Medjed is a good example of this, with the foothold exploit leveraging several seemingly unrelated web apps. While not too difficult, it’s very easy to get stuck without thorough enumeration.

### TCP8000 and TCP44330 - BarracudaDrive:

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_bd_landing.png)

Port 8000 is running BarracudaDrive (now renamed as FuguHub), a Lua-based application that offers features like file sharing, lightweight application hosting, and CMS. This is something OffSec loves putting in their boxes: clunky and legacy management software with horrible user experience, but often running with elevated privileges and packed with abusable built-in features.

After a few seconds, it redirected to `/Config-Wizard/wizard/SetAdmin.lsp` prompting me to setup  an admin user.

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_bd_newadmin.png)

This indicated it’s a fresh install with no users configured yet. I’ll create a new admin `ch3ng`.

The about page also showed its exact version.

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_bd_about.png)

This version is ancient. According to its [changelog](https://fuguhub.com/readme.txt), 6.5 was released in May 2013! Searchsploit found a [public exploit](https://www.exploit-db.com/exploits/48789) for this exact version.

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_bd_webfileserver.png)

### TCP33033 - Rails App:

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_rails_landing.png)

Port 33033 is a Rails app. The landing page simply lists all the users and their profiles. There doesn’t seem to be any naming convention, with multiple username formats found. Jerren appears to be the sysadmin, based on his username `jerren.devops`.

The button at the top right leads to a login page.

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_rails_login.png)

The login form does not seem injectable, and it does not return any error messages either.

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_rails_forgot.png)

There’s also a password reset function, which requires entering a reminder phrase. I tried with `admin` but failed. The error message did however tell me that the user does not exist.

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_rails_admin_fail.png)

For example, `joe.webb` gives a different error:

![](https://raw.githubusercontent.com/ch3ng625/blog_images/refs/heads/main/medjed/web_rails_joe_fail.png)

Other than that, there’s not much more to look at. Since I don’t see any paths forward here, I’ll review what feroxbuster found in the background.

_NOTE: According to the official walkthrough, the intended path here is to look at Jerren’s profile, guess his reminder phrase is “paranoid” to reset his password and log in to the app. I didn’t figure this out when I worked on the box, so I did it another way. I find the guessing part rather far-fetched, and having this as the intended path seems somewhat questionable in my opinion._

Turns out I misunderstood what “Disabled” means. All the listed privileges have already been granted to the user, “disabled” just means the privilege is not active in the current session.
